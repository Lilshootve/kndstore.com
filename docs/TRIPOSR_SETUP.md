# TripoSR / InstantMesh 3D Setup

## Overview

The 3D integration allows users to upload an image and receive a 3D model (OBJ/GLB) generated by an external GPU server. The backend can use **TripoSR** or **InstantMesh**; the API contract is the same. KND uses `TRIPOSR_API_URL` and `TRIPOSR_CALLBACK_SECRET` for compatibility.

## Database

Run the migration:

```sql
source sql/triposr_jobs.sql
```

## Configuration

1. Copy the example config:
   ```
   cp config/triposr_secrets.local.example.php config/triposr_secrets.local.php
   ```

2. Edit `config/triposr_secrets.local.php`:
   - `TRIPOSR_API_URL`: Base URL of your GPU server `/generate` endpoint (e.g. `https://gpu.example.com/generate` for InstantMesh)
   - `TRIPOSR_CALLBACK_SECRET`: Shared secret for callback validation (generate with `openssl rand -hex 32`)
   - `TRIPOSR_UPLOAD_DIR`: `triposr/uploads` (relative to storage/)
   - `TRIPOSR_OUTPUT_DIR`: `triposr/outputs` (relative to storage/)

3. Add to `includes/config.php` (optional, for early loading):
   ```php
   require_once __DIR__ . '/triposr_config.php';
   ```
   If omitted, the API endpoints will load it automatically.

## GPU Server Contract

Your GPU server (TripoSR or InstantMesh) must:

1. **Accept POST** at `TRIPOSR_API_URL` with JSON:
   ```json
   {
     "job_id": "uuid",
     "image_url": "https://yoursite.com/api/triposr/image.php?t=uuid",
     "callback_url": "https://yoursite.com/api/triposr/callback.php",
     "secret": "shared_secret"
   }
   ```

2. **Fetch the image** from `image_url` (no auth, UUID in `t` is unguessable).

3. **POST to callback_url** when done:
   ```json
   {
     "job_id": "uuid",
     "status": "completed",
     "model_url": "https://gpu-server/path/to/model.glb",
     "secret": "shared_secret"
   }
   ```
   Or on failure:
   ```json
   {
     "job_id": "uuid",
     "status": "failed",
     "error_message": "Optional error",
     "secret": "shared_secret"
   }
   ```

4. The KND backend downloads the model from `model_url` and saves it to storage.

## Storage

- Uploads: `storage/triposr/uploads/{uuid}.jpg` (or .png, .webp)
- Outputs: `storage/triposr/outputs/{job_uuid}.glb` or `.obj`

Ensure the web server can write to these directories.

## GPU Server (InstantMesh)

See `gpu-server/README_DEPLOY.md` for the InstantMesh GPU server implementation (FastAPI, Docker, RunPod/Vast). The server exposes:

- `POST /generate` — enqueue job, return 202
- `GET /models/{job_id}.glb` — serve generated model
- `GET /health` — health check

Set `TRIPOSR_API_URL` to `https://your-gpu-host/generate`.
